# 如何在手机上查看服务器日志

## 方法1：通过Web界面查看（推荐）✨

1. **登录账号**
   - 在手机上打开 openEar 网站
   - 登录你的账号

2. **访问日志页面**
   - 点击导航栏右上角的 **"📋 日志"** 链接
   - 或者直接访问：`http://你的服务器地址/logs`

3. **查看日志**
   - 页面会显示服务器日志的最后1000行
   - 点击 **"🔄 刷新"** 按钮可以更新日志
   - 日志会自动滚动到底部（最新内容）

## 方法2：通过SSH客户端（需要服务器访问权限）

### iOS 推荐应用：
- **Termius**（免费，界面友好）
- **Blink Shell**（付费，功能强大）
- **Prompt 3**（付费）

### Android 推荐应用：
- **JuiceSSH**（免费，功能全面）
- **Termux**（免费，开源）
- **ConnectBot**（免费，开源）

### 使用步骤：
1. 安装SSH客户端应用
2. 连接到服务器：
   ```
   ssh 用户名@服务器IP地址
   ```
3. 查看日志：
   ```bash
   # 查看实时日志（Gunicorn）
   tail -f logs/app.log
   
   # 查看最后100行
   tail -n 100 logs/app.log
   
   # 查看完整日志
   cat logs/app.log
   
   # 搜索特定内容
   grep "错误" logs/app.log
   ```

## 方法3：查看服务器控制台输出

如果服务器正在运行，可以直接查看控制台输出：
- **开发环境**：直接在运行 `python app.py` 的终端查看
- **生产环境**：查看 Gunicorn 的输出或系统日志

## 日志文件位置

- **开发环境**：日志通常直接输出到控制台
- **生产环境**：`logs/app.log` 或 `openEar/logs/app.log`

## 常见问题

### Q: 日志页面显示"未找到日志文件"
A: 
- 确保服务器正在运行
- 检查日志文件是否存在：`ls -la logs/app.log`
- 如果使用 Gunicorn，确保日志配置正确

### Q: 如何查看实时日志？
A: 
- Web界面：点击"刷新"按钮
- SSH：使用 `tail -f logs/app.log` 命令

### Q: 日志文件太大怎么办？
A: 
- 使用 `tail -n 100 logs/app.log` 只查看最后100行
- 或者使用日志轮转工具（如 logrotate）
